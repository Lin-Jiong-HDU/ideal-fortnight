# 阶段 1：核心工单流程设计

## 概述

GEO 平台阶段 1 实现：登录认证 + 工单创建/领取/处理/交付 + 客户审核 + 基础 Dashboard。

## 技术方案

| 选项 | 选择 |
|------|------|
| 框架 | Next.js 16 + React 19 |
| 样式 | Tailwind CSS 4 |
| UI 组件 | 纯 Tailwind 自定义组件 |
| 状态管理 | React Context |
| 数据获取 | 原生 fetch |
| 认证 | JWT + localStorage |

## 设计风格

- **风格**：现代 SaaS 风（Vercel/Stripe 风格）
- **特点**：圆角卡片、渐变、更多视觉元素
- **布局**：侧边栏导航

## 页面路由

| 路由 | 说明 | 权限 |
|------|------|------|
| `/login` | 登录页 | 公开 |
| `/dashboard` | 首页 | 登录用户 |
| `/tickets` | 工单列表 | 登录用户 |
| `/tickets/new` | 创建工单 | admin/optimizer/customer |
| `/tickets/:id` | 工单详情 | 登录用户 |

## 工单状态流转

```
pending → assigned → processing → reviewing → completed
                                    ↓
                              (rejected)
                                    ↓
                              processing
```

## 状态颜色

| 状态 | 颜色 | 文案 |
|------|------|------|
| pending | 灰色 | 待领取 |
| assigned | 蓝色 | 已分配 |
| processing | 橙色 | 处理中 |
| reviewing | 紫色 | 待审核 |
| completed | 绿色 | 已完成 |

## 后端 API 依赖

阶段 1 需要调用的后端接口：

- `POST /login` - 登录
- `GET /tickets` - 获取工单列表（customer 角色只能看自己的）
- `POST /tickets` - 创建工单
- `GET /tickets/:id` - 获取工单详情
- `POST /tickets/:id/review` - 审核工单（customer）
- `GET /optimizer/tickets` - 获取待处理工单池
- `POST /optimizer/tickets/:id/claim` - 领取工单
- `POST /optimizer/tickets/:id/process` - 开始处理
- `PUT /optimizer/tickets/:id/result` - 提交结果
- `POST /optimizer/tickets/:id/deliver` - 交付工单

## 验收标准

1. 三种角色可以登录并看到对应的导航菜单
2. Admin/Optimizer 可以创建工单、领取工单、提交结果、交付
3. Customer 可以查看自己的工单、审核工单
4. Dashboard 显示正确的统计数据
5. 工单详情页显示评分对比和内容
